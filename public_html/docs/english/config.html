glFusion Classifieds Plugin Configuration

The Classified Ads Plugin allows your site users to create classified ads. You create categories and subcategories to classify the ads, and can control publication by using the submission queue.

General Options

Max Image Height
Maximum height, in pixels, for images. Uploaded images will be scaled to the maximum width/height, preserving the aspect ratio.
Max Image Width
Maximum width, in pixels, for images. Uploaded images will be scaled to the maximum width/height, preserving the aspect ratio.
Detail Page Image Width
Maximum width of images shown at the left of the ad detail pages. The height is flexible and will be automatically calculated.
Max Thumb Size
Maximum dimension for thumbnails. Upload images will be scaled to this size, preserving the aspect ratio, to create thumbnails.
Random Block Image Width
Maximum width for images displayed in the "Random Ad" block. The image will be scaled to preserve the aspect ratio. This setting is to allow the administrator to use wider-than-normal blocks.
Max. number of images
This is the maximum number of images that users may upload to a single ad.
Use submission queue
Set this to "True" if user ads should be moderated. If this is "False", ads will be publically available as soon as they are submitted.
Default Duration
Default number of days before an ad expires. This can be overridden by the user when the ad is created, and additional days can be added to the ad later.
Number of days for a category to be considered "new"
The number of days after a category is added that it will show with the "new" flag.
Number of days for an ad to be considered "new"
The number of days after an ad is created that it will show up in the "New Ads" block.
Hide the "New Ads block
Select whether the plugin will hide new ads from the "What's New" block.
  • Yes - Don't show anything in "What's New".
  • No - Always show something. If there are no new ads, a message to that effect will be displayed.
  • If Nothing New - Suppress the "What's New" display if there are no new ads.
Email the admin when an ad is submitted
If set to "Always", the site admin is always notified of a new ad submission. If set to "If Submission Queue", then the site admin will receive notification only if the ads are moderated. If set to "Never", the site admin will never receive notifications.
Email the user upon acceptance?
An email message can be sent to the submitter after moderation when an add is accepted, rejected, either, or never. This setting has no effect if the submission queue is not used.
Hide this plugin from the User Functions menu
Set to "Yes" to keep this plugin from appearing on the "User Functions" menu.
Max ads shown per page (expanded listing)
The maximum number of ads to be shown in the expanded listing. This is the list which is shown under the "Recent" and category links, and includes detailed information about the ad.
Max ads shown per page (listing)
The maximum number of ads to be shown in the compressed listing. This is the list which shows the ad subject on one line, typically with links to edit, approve, and delete the ad.
Maximum total ad duration, in days
The maximum number of days that an ad may run. This number limits the amount of time that an ad may be extended.
Days after expiration to purge ads
If this number is greater than zero, then ads which have expired more than this many days ago will be purged from the ad table. Set to zero to disable purging.
Days before expiration to send reminders
Enter the number of days prior to an ad's expiration date for a reminder to be sent to the ad owner. The reminder is only sent to owners who have requested them by updating their account preferences.

Set to "-1" to disable reminders completely. Setting to "0" causes a message to be sent immediately upon expiration.

Login required for all access?
If set to "Yes", users must login to view or search ads. User login is always required to submit ads.
Show owner full naem with ads?
Select "e;Yes" to display the ad owner's full name with the ad. "No" indicates that only the login name should be shown.
Users can edit their own ads?
If set to "Yes", users can edit their own ads after approval. Setting to "No" disables editing. If ads are moderated, this should be set to "No"
Use glFusion Task Scheduler?
If set to "Yes", then the glFusion task scheduler is used to purge old ads and send expiration notifications. If you'd like more control over when these tasks are performed, then set this field to "No" and use the included standalone cron.php (located in <plugin_dir>/classifieds/utility) with your server's cron facility.
Category Display Mode
Determines how the main categories are displayed on the home page.
  • Normal - Categories are displayed as text items, with sub-category links listed under them. This is the zClassifieds method.
  • Blocks- Main categories are shown as blocks, with the icon and descriptive text within the colored block. The admin may specify foreground and background colors, or random system defaults will be used. Does not work properly with IE7 or earlier
Replace Home Page?
If set to "Yes", then the site home page will be replaced by the Classified Ads plugin. Leave this as "No" for the site to use its normal home page.
Comments Supported?
Set this to "Yes" to allow comments to be added to ads.
Display glFusion Blocks
Select which blocks will be displayed along with the ad listing and detail pages.

Permissions

Default Ad Group
This is the default group which will be able to view ads. This can be changed during ad submission.
Default Category Group
This is the default group which will be assigned to the "group_id" value for newly-created categories. The "Category Default Permission[1]" selected will be applied as well to give viewing and submission permission to the category.

For categories, permissions have the following meanings:

  • None: Users may not view ads under the category.
  • Read-only: Users may view ads under the category.
  • Read-Write: Users may view and submit ads to the category.
Anonymous users may never submit ads, so "read-write" permission only grants "read" permission to them.
